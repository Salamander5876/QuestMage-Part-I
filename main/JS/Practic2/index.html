Практическая работа по закреплению JS

1. Калькулятор

Техническое задание (ТЗ):

  • Создать веб-страницу с калькулятором, который позволяет выполнять базовые арифметические операции: сложение, вычитание, умножение, деление.
  • Калькулятор должен иметь кнопки для цифр от 0 до 9, кнопки для операций (+, -, *, /), кнопку равно (=), кнопку очистки (C).
  • При нажатии на кнопки цифр и операций, на экране калькулятора должно отображаться текущее выражение.
  • При нажатии на кнопку "=", на экране должен отображаться результат вычисления.
  • Обработать возможные ошибки, например, деление на ноль.

Подсказки:

  • Используйте HTML для создания структуры калькулятора: кнопок и поля для вывода (например, <input type="text"> или <div>).
  • Используйте CSS для стилизации калькулятора, чтобы он выглядел как настоящий (например, сетка кнопок с помощью display: grid).
  • В JavaScript создайте функции для обработки нажатий на кнопки с помощью addEventListener.
  • Для хранения текущего выражения используйте строку, а для вычисления результата можно применять функцию eval() (но осторожно, либо пишите свой парсер).
  • При нажатии на кнопку "=" вычисляйте результат и показывайте его на экране.
  • Для кнопки "C" очищайте строку выражения и обновляйте экран.

2. Крестики-нолики

Техническое задание (ТЗ):

  • Создать веб-страницу с игрой "Крестики-нолики" для двух игроков.
  • Игровое поле — сетка 3x3 клетки.
  • Игроки по очереди ставят свои знаки (X или O) в пустые клетки.
  • Игра заканчивается, когда один из игроков выстраивает три своих знака в ряд (по горизонтали, вертикали или диагонали), или когда все клетки заполнены (ничья).
  • Отображать текущий ход (чей ход) и результат игры (победа или ничья).

Подсказки:

  • Используйте HTML для создания игрового поля: например, <table> или сетку из <div> с помощью CSS grid.
  • Стилизуйте поле и знаки (X и O) с помощью CSS.
  • В JavaScript храните состояние игры в массиве 3x3, где каждая ячейка — это пустое значение, "X" или "O".
  • При клике на клетку проверяйте, свободна ли она, и если да, добавляйте знак текущего игрока.
  • После каждого хода проверяйте выигрышные комбинации (8 вариантов: 3 строки, 3 столбца, 2 диагонали) или ничью.
  • Обновляйте текст на странице, чтобы показывать, чей ход или кто победил.

3. Змейка

Техническое задание (ТЗ):

  • Создать веб-страницу с игрой "Змейка".
  • Змейка движется по полю, ест еду и увеличивается в размере.
  • Игра заканчивается, если змейка врезается в стену или в саму себя.
  • Управление змейкой осуществляется с помощью клавиш стрелок.
  • Отображать текущий счёт (количество съеденной еды).

Подсказки:

  • Используйте HTML5 <canvas> для рисования поля или создайте сетку из <div> с помощью CSS.
  • В JavaScript создайте массив или объект для хранения координат тела змейки и её направления.
  • Используйте setInterval, чтобы змейка двигалась автоматически с заданной скоростью.
  • Обрабатывайте нажатия клавиш с помощью addEventListener на событие keydown для изменения направления.
  • Генерируйте еду в случайной клетке поля с помощью Math.random().
  • При столкновении с едой увеличивайте длину змейки и счёт, при столкновении со стеной или собой завершайте игру.

4. Пятнашки

Техническое задание (ТЗ):

  • Создать веб-страницу с игрой "Пятнашки".
  • Игровое поле — сетка 4x4 с номерами от 1 до 15 и одной пустой клеткой.
  • Игрок может перемещать соседние клетки в пустое место, чтобы собрать последовательность от 1 до 15.
  • Игра заканчивается, когда все клетки расположены в правильном порядке.

Подсказки:

  • Используйте HTML для создания поля: например, <table> или сетку из <div> с CSS grid.
  • В JavaScript храните состояние поля в массиве 4x4, где одна ячейка — пустая (например, 0 или null).
  • При клике на клетку проверяйте, соседствует ли она с пустой клеткой, и если да, меняйте их местами.
  • Используйте addEventListener для обработки кликов по клеткам.
  • После каждого хода проверяйте, соответствует ли текущее состояние правильной последовательности (1–15).
  • Обновляйте DOM, чтобы отображать изменения на экране.

5. Сапёр

Техническое задание (ТЗ):

  • Создать веб-страницу с игрой "Сапёр".
  • Игровое поле — сетка (например, 9x9), в некоторых клетках скрыты мины.
  • Игрок открывает клетки, стараясь не открыть мину.
  • Если открыта клетка с миной, игра проиграна.
  • Если открыта клетка без мины, показывается количество мин в соседних клетках.
  • Игра выиграна, когда открыты все клетки без мин.

Подсказки:

  • Используйте HTML для создания сетки: например, <table> или <div> с CSS grid.
  • В JavaScript создайте двумерный массив для хранения мин и состояния клеток (открыта/закрыта).
  • Сгенерируйте мины случайным образом (например, 10 мин для поля 9x9) при старте игры.
  • При клике на клетку открывайте её: если это мина — завершайте игру, если нет — показывайте число соседних мин.
  • Используйте рекурсию, чтобы автоматически открывать соседние пустые клетки, если мин рядом нет.
  • Обновляйте DOM для отображения открытых клеток и чисел.

Эти пять тем — калькулятор и четыре игры — разнообразны по сложности и механике. Они позволяют вам практиковать работу с DOM, обработку событий, управление состоянием и базовую логику программирования на JavaScript. Выбирайте любую тему в зависимости от ваших интересов!
